这篇文章想回答的问题是：AI 编程的效率，到底来自提示词技巧，还是来自工程化的目标定义和执行环境？

我暂时的判断是，真正重要的不是“会不会和模型聊天”，而是能否把人的意图转化成 Agent 可以执行、验证、迭代和回滚的任务系统。换句话说，AI 编程不是把代码外包给一个更聪明的自动补全，而是在做一种更接近软件项目管理的 Agentic Software Engineering。

## 问题起点：Coding Agent 不等于 ChatGPT 聊天框

- 我之前容易把 Codex 的 Plan Mode 当成 ChatGPT 网页端使用。
- 但 Coding Agent 更偏 action：编辑文件、运行命令、生成补丁、验证结果。
- ChatGPT 更适合开放讨论、概念澄清、路线比较和认知建模。
- 因此，一个更合理的流程可能是：先用 ChatGPT 完成问题建模和任务边界定义，再把明确任务交给 Coding Agent 执行。

## AI 编程的两个核心变量：目标与 Harness

目标定义解决“做什么”：

- 最终要交付哪些 artifacts？
- 代码、测试、文档、配置、迁移说明、验收标准分别是什么？
- 哪些内容明确不做？

Harness 解决“怎么持续做下去”：

- 工作目录、文件权限、CLI、MCP 接口；
- 项目文档、AGENTS.md / CLAUDE.md / Codex 指令；
- 构建命令、测试命令、危险操作边界；
- 可访问上下文、回滚机制、验证闭环。

一个初步公式：

```text
AI Coding 效率 = 目标清晰度 × Harness 完整度 × 任务粒度匹配度
```

## 三种协作模式：手动编码、Vibe Coding 与 SDD

| 协作方式 | 适合粒度 | 典型场景 |
| --- | --- | --- |
| 手动编码 | WBS 底层 activity / task | 几行配置、明确 bug、小范围替换 |
| Vibe Coding / 轻量 Agent Coding | WBS 三级子工作包或较大 task | 脚本、小功能、局部重构、探索式 demo |
| SDD | WBS 二级工作包，甚至小项目一级范围 | 完整模块、明确验收、高交付要求任务 |

这里的关键不是“AI 参与越多越好”，而是 AI 参与程度必须和任务粒度匹配。当同步目标和构建 harness 的成本高于自己动手时，手动编码反而是更工程化的选择。

## SDD 不是多写文档，而是 Agent 控制面

我不想把 SDD 理解成“写很多文档”。更准确地说，它是一种 Agent 控制系统。

SDD 的文档至少服务于四件事：

- 目标固化：防止执行过程中目标漂移；
- 过程追踪：知道 Agent 为什么这样改、改了什么、进行到哪一步；
- 验收判断：让人和 Agent 都知道什么叫完成；
- 回滚与恢复：改坏时可以根据计划、记录和提交回到可靠状态。

也就是说，文档不是附属物，而是 Agent 执行过程的人类可追溯、可干预、可回滚的控制面。

## PRD、Scope Statement 与 WBS 的位置

PRD 可以承担 SDD 中“目标声明”的核心作用，但它不完全等同于项目管理里的范围说明书。

| Artifact | 更关注什么 |
| --- | --- |
| PRD | 用户场景、功能需求、交互逻辑、验收标准 |
| Scope Statement | 交付边界、不交付内容、约束、假设、验收条件 |
| RTM | 需求如何映射到设计、开发、测试和验收 |
| WBS | 把范围拆成可管理的工作包 |

因此，在 Agent 驱动开发里，PRD 最好和 Spec、WBS、验收标准、Test Plan、Agent 执行约束一起使用，而不是单独承担所有上下文。

## 任务粒度之外：还要看不确定性

任务粒度不是唯一维度。另一个关键变量是目标是否稳定。

| 任务情况 | 推荐方式 |
| --- | --- |
| 小而确定 | 手动编码 |
| 小而不确定 | 先讨论，再手动改 |
| 中等且不确定 | Vibe Coding / 探索式 Agent Coding |
| 中等且确定 | Vibe Coding 或小型 SDD |
| 大而确定 | SDD |
| 大而不确定 | 先方案探索和范围澄清，再拆成 SDD 工作包 |

SDD 最适合大粒度、高确定性、高交付要求的任务。如果目标还很模糊，过早 SDD 可能只是把错误方向文档化。

## 暂定结论

AI 编程不是“让模型写代码”，而是“基于任务粒度选择合适的人机协作模式”。

对于微小精确任务，手动修改最快；对于中等复杂、目标尚可探索的任务，使用 Vibe Coding；对于高复杂度、需要交付完整制品的工作包，使用 SDD，通过详细目标定义、完整 harness 和文档化追踪来驱动 Agent 执行。

最后可以浓缩成一句话：

> 把 AI 编程从“对话式写代码”升级为“基于范围管理、制品定义、harness 构建和可追溯执行的工程化 Agent 驱动开发”。
